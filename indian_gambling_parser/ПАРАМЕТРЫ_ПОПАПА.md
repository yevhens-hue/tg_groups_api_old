# Параметры, вводимые в попапе

## Данные из конфига

Парсер использует данные из `merchants_config.py`, секция `provider_form_data`:

```python
"provider_form_data": {
    "upi_id": "kapil.ss17@icici",
    "phone_number": "+918925242020",
    "aadhaar": "571517312306",
}
```

---

## Логика заполнения

**Важно**: Парсер заполняет **только ОДНО поле** в попапе - то, которое первое найдётся.

### Приоритет заполнения:

1. **UPI ID** (приоритет 1, самый высокий)
   - Значение: `kapil.ss17@icici`
   - Поиск поля:
     - По label с текстом "upi id" (регистронезависимо, regex)
     - По input с атрибутами: `name*='upi'`, `placeholder*='upi'`, `id*='upi'`
   - Если поле найдено и видимо → заполняется и остальные поля пропускаются

2. **Phone Number** (приоритет 2, только если UPI ID не найден)
   - Значение: `+918925242020`
   - Поиск поля:
     - По label с текстом "number|phone|mobile" (regex, регистронезависимо)
     - По input с атрибутами: `type='tel'`, `name*='phone'`, `placeholder*='phone'`, `id*='phone'`
   - Если поле найдено и видимо → заполняется, Aadhaar пропускается

3. **Aadhaar** (приоритет 3, только если UPI ID и Phone не найдены)
   - Значение: `571517312306`
   - Поиск поля:
     - По label с текстом "aadhaar" (регистронезависимо, regex)
     - По input с атрибутами: `name*='aadhaar'`, `placeholder*='aadhaar'`, `id*='aadhaar'`
   - Если поле найдено и видимо → заполняется

---

## Алгоритм поиска полей

Для каждого типа поля (UPI ID, Phone, Aadhaar) парсер использует два метода:

### Метод 1: Поиск по label (приоритет выше)
```python
deposit_frame.get_by_label(re.compile(r"upi\s*id", re.I))
deposit_frame.get_by_label(re.compile(r"number|phone|mobile", re.I))
deposit_frame.get_by_label(re.compile(r"aadhaar", re.I))
```

### Метод 2: Поиск по input атрибутам (fallback)
```python
# UPI ID
input[name*='upi' i], input[placeholder*='upi' i], input[id*='upi' i]

# Phone Number
input[type='tel'], input[name*='phone' i], input[placeholder*='phone' i], input[id*='phone' i]

# Aadhaar
input[name*='aadhaar' i], input[placeholder*='aadhaar' i], input[id*='aadhaar' i]
```

## Примеры работы

### Пример 1: Попап с полем UPI ID
1. Парсер находит поле "UPI ID"
2. Заполняет значение: `kapil.ss17@icici`
3. **НЕ ищет** Phone Number и Aadhaar (так как уже заполнил UPI ID)
4. Нажимает "ПОДТВЕРДИТЬ"

### Пример 2: Попап с полем Phone Number (без UPI ID)
1. Парсер ищет поле "UPI ID" → не находит
2. Парсер находит поле "Phone Number" или "Mobile Number"
3. Заполняет значение: `+918925242020`
4. **НЕ ищет** Aadhaar (так как уже заполнил Phone Number)
5. Нажимает "ПОДТВЕРДИТЬ"

### Пример 3: Попап с полем Aadhaar (без UPI ID и Phone)
1. Парсер ищет поле "UPI ID" → не находит
2. Парсер ищет поле "Phone Number" → не находит
3. Парсер находит поле "Aadhaar"
4. Заполняет значение: `571517312306`
5. Нажимает "ПОДТВЕРДИТЬ"

### Пример 4: Попап без полей или поля уже заполнены
1. Парсер ищет все три типа полей → ничего не находит
2. Выводит: "Поля не найдены или уже заполнены, продолжаем"
3. Нажимает "ПОДТВЕРДИТЬ" (без заполнения)

---

## Безопасность

- ✅ Значения **не хардкодятся** в коде, берутся из конфига
- ✅ Значения **не логируются** в консоль (выводится только "✓ Заполнено поле UPI ID" без значения)
- ⚠️ Значения **сохраняются в скриншотах** (можно добавить маскирование опционально)

## Изменение параметров

Чтобы изменить значения, нужно отредактировать файл:
`indian_gambling_parser/merchants_config.py`

Секция для 1xbet:
```python
"provider_form_data": {
    "upi_id": "kapil.ss17@icici",      # ← Изменить здесь
    "phone_number": "+918925242020",   # ← Изменить здесь
    "aadhaar": "571517312306",         # ← Изменить здесь
}
```

После изменения значений нужно перезапустить парсер.
