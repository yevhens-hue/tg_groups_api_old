# 📚 API Версионирование

**Дата:** 2026-01-11  
**Версия API:** 1.1.0

---

## 🎯 Обзор

API поддерживает версионирование для обеспечения обратной совместимости и плавного развития.

---

## 📋 Структура версий

### Текущие версии

- **v1**: `/api/v1/*` - Стабильная версия 1
- **latest**: `/api/*` - Текущая версия (совместима с v1)

---

## 🔗 Endpoints

### v1 Endpoints

#### Providers

- `GET /api/v1/providers` - Получить список провайдеров
- `GET /api/v1/providers/{id}` - Получить провайдера по ID
- `PUT /api/v1/providers/{id}` - Обновить провайдера

**Пример запроса:**
```bash
curl http://localhost:8000/api/v1/providers?merchant=1xbet&limit=10
```

**Пример ответа:**
```json
{
  "total": 100,
  "items": [...],
  "skip": 0,
  "limit": 10,
  "version": "v1"
}
```

---

## 🔄 Миграция между версиями

### Использование v1

Все endpoints v1 возвращают поле `version: "v1"` в ответе для идентификации версии.

### Обратная совместимость

Endpoints без версии (`/api/providers`) совместимы с v1 и работают аналогично.

---

## 📝 Best Practices

### Рекомендации

1. **Используйте версионированные endpoints** для production
2. **Указывайте версию в запросах** для стабильности
3. **Проверяйте поле `version`** в ответах
4. **Мигрируйте постепенно** при появлении новых версий

### Примеры

```python
# ✅ Хорошо: явное указание версии
response = requests.get("http://api.example.com/api/v1/providers")

# ⚠️ Можно: использование latest (может измениться)
response = requests.get("http://api.example.com/api/providers")
```
