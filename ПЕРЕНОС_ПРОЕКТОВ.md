# 📦 Инструкция по переносу проектов на другой компьютер

## Метод 1: Архивация через tar (рекомендуется)

### Шаг 1: Создание архива с проектами

```bash
cd /Users/yevhen.shaforostov

# Создаём архив со всеми проектами (исключая системные папки)
tar -czf projects_backup_$(date +%Y%m%d_%H%M%S).tar.gz \
  indian_gambling_parser/ \
  mirror_finder/ \
  mirrors_api/ \
  conductor/ \
  1win/ \
  1win_ar/ \
  tg_api_old/ \
  tg_groups_api_old/ \
  --exclude='*/__pycache__' \
  --exclude='*/.git' \
  --exclude='*/node_modules' \
  --exclude='*/venv' \
  --exclude='*/env' \
  --exclude='*/.DS_Store' \
  --exclude='*/screenshots/*.png' \
  --exclude='*/traces/*.zip' \
  --exclude='*/storage_states/*.json' \
  --exclude='*/providers_data.db' \
  --exclude='*/providers_data.xlsx' \
  --exclude='*/notification_*.txt' \
  --exclude='*/test_run_*.log'
```

### Шаг 2: Перенос архива

**Вариант A: Через внешний диск / USB**
```bash
# Скопируйте архив на внешний диск
cp projects_backup_*.tar.gz /Volumes/YourUSB/
```

**Вариант B: Через сеть (rsync/scp)**
```bash
# Если есть SSH доступ к другому компьютеру
scp projects_backup_*.tar.gz user@new-computer:/path/to/destination/
```

**Вариант C: Через облачное хранилище**
```bash
# Загрузите на Google Drive / Dropbox / iCloud
# Или используйте rclone / rsync к облачному хранилищу
```

### Шаг 3: Распаковка на новом компьютере

```bash
# Распакуйте архив
tar -xzf projects_backup_*.tar.gz

# Переместите проекты в нужную директорию
mv indian_gambling_parser ~/
mv mirror_finder ~/
# и т.д.
```

## Метод 2: rsync (синхронизация через сеть)

### Если у вас есть SSH доступ к новому компьютеру

```bash
# Синхронизация всех проектов
rsync -avz --progress \
  --exclude='__pycache__' \
  --exclude='.git' \
  --exclude='node_modules' \
  --exclude='venv' \
  --exclude='env' \
  --exclude='.DS_Store' \
  --exclude='screenshots/*.png' \
  --exclude='traces/*.zip' \
  --exclude='storage_states/*.json' \
  --exclude='providers_data.db' \
  --exclude='providers_data.xlsx' \
  --exclude='notification_*.txt' \
  --exclude='test_run_*.log' \
  /Users/yevhen.shaforostov/indian_gambling_parser/ \
  /Users/yevhen.shaforostov/mirror_finder/ \
  /Users/yevhen.shaforostov/mirrors_api/ \
  user@new-computer:/path/to/destination/
```

## 📋 Чеклист после переноса на новый компьютер

### 1. Установка зависимостей Python

```bash
# Для каждого проекта
cd indian_gambling_parser
pip3 install -r requirements.txt

cd ../mirror_finder
pip3 install -r requirements.txt

cd ../mirrors_api
pip3 install -r requirements.txt
```

### 2. Установка Playwright браузеров

```bash
cd indian_gambling_parser
python3 -m playwright install chromium
```

### 3. Проверка конфигурации

- Проверить пути к файлам в конфигах
- Проверить наличие `google_credentials.json` (если нужен)
- Обновить пути в документации (если изменилась структура)

### 4. Тестирование

```bash
# Проверка импортов
python3 -c "from storage import Storage; print('OK')"

# Проверка синтаксиса
python3 -m py_compile *.py
```

## 🗑️ Что НЕ нужно переносить (можно исключить)

- `__pycache__/` - скомпилированные Python файлы
- `.git/` - Git репозиторий (если используете Git, клонируйте отдельно)
- `node_modules/` - зависимости Node.js (установятся через npm)
- `venv/`, `env/` - виртуальные окружения (создадите заново)
- `.DS_Store` - системные файлы macOS
- `screenshots/*.png` - скриншоты (можно не переносить)
- `traces/*.zip` - trace файлы Playwright
- `storage_states/*.json` - сохранённые сессии (можно пересоздать)
- `providers_data.db`, `providers_data.xlsx` - данные (сгенерируются заново)
- `notification_*.txt`, `test_run_*.log` - временные логи
