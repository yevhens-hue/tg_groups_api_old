# Резюме отладки парсера 1win AR

## Проблема

После клика на кнопку "Deposit" происходит редирект на промо-страницу (`https://1win.lat/promotions?trid=banner1wBonusUpTo600INbr`), и модальное окно с методами оплаты не открывается. В результате парсер не может найти Claro Pay и извлечь данные (CVU, Recipient, Amount).

## Попытки решения

### 1. ✅ Перехват навигации через `page.route()`
- **Статус:** Не сработало
- **Причина:** Редирект происходит через JavaScript (`window.location`), а не через HTTP-запросы

### 2. ✅ Перехват через `framenavigated` событие
- **Статус:** Не сработало
- **Причина:** Событие срабатывает после редиректа, возврат назад не помогает

### 3. ✅ Перехват `window.location` и `history.pushState/replaceState` через JavaScript
- **Статус:** Не сработало
- **Причина:** Proxy для `window.location` не работает, так как это не обычный объект

### 4. ✅ Поиск модального окна через JavaScript в DOM
- **Статус:** Не сработало
- **Причина:** Модальное окно не существует в DOM на промо-странице

### 5. ✅ Поиск функций открытия модального окна
- **Статус:** Не сработало
- **Причина:** Функции не найдены в глобальном scope

### 6. ✅ Попытка вернуться назад с промо-страницы
- **Статус:** Не сработало
- **Причина:** После возврата назад снова происходит редирект

### 7. ✅ JS-клик на кнопку Deposit с `preventDefault`
- **Статус:** Не сработало
- **Причина:** Редирект происходит не через событие клика, а через другой механизм

### 8. ✅ Прямое открытие модального окна через изменение стилей
- **Статус:** Не сработало
- **Причина:** Модальное окно не существует в DOM на промо-странице

### 9. ✅ Поиск Claro Pay на всей странице через JavaScript
- **Статус:** Не сработало
- **Причина:** Claro Pay не существует на промо-странице
